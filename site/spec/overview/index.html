Overview

Here you'll find the basic concepts needed before diving into the world of JSON Schema

Introduction

Before understanding how to define JSON Schemas and how they work, it is required to understand what a JSON document is. If you already know what a JSON document is you can jump directly to the JSON Schema section.

JSON

JSON (JavaScript Object Notation) is a file format commonly used for sharing information over the Web. JSON files are lightweight and easy to ready by both machines and developers. This has made JSON the most popular file format for client-server communication. A JSON document is one of the next seven elements:

string number integer boolean null array object

-String

A string is simply a sequence of valid characters between quotes. For example, "Hello world!" is a string, but 23 isn't.

-Number

A number in JSON is any real number using floating-point or exponential notation. For example -2, 23, 3.14 and 2.54e3 are all numbers in JSON. The full specification can be found here.

-Integer

All the numbers without a fractional part are integers. Integers are thus a subset of numbers. For example, 34 and -91 are integers.

-Boolean

A boolean JSON object is one of two values: true or false.

-Null

A null object in JSON is the value null.

-Array

A JSON array is an ordered sequence of JSON documents separated by comma and enclosed by brackets. For example, ["Hello", 23, true, null] is a JSON array, as well as [-2, -1, 3.14, 1.62e10].

-Object

A JSON object is a set of pairs of the form key:value, where key is a JSON string and value is a JSON document. This pairs are separated by comma and enclosed by braces. JSON objects are the most common JSON documents. The next is an example of a JSON object storing information about the famous Chilean football player Gary Medel:

{
    "first_name": "Gary",
    "last_name": "Medel",
    "age": 27,
    "club": {
        "name": "Inter Milan",
        "founded": 1908
    } 
}

JSON Schema

A JSON Schema is a document that specifies the structure of the JSON documents that are used for a certain application. For example, in a store, one might want to share documents with the information of products. Here, one might specify that each of such documents is a list of products, and that each product contains a name, an id, a stock and optionally a link to the manufacturer's website. To correctly understand how JSON Schemas work, it is first required to understand what a JSON document is.
